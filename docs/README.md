# Tower Brawlers Documentation

This folder contains the structured pre-production docs for `Tower Brawlers: A Soup-to-Nuts War`, a silly parody lane-war strategy game with comedic factions, absurd stakes, and touch-friendly RTS-lite gameplay.

## Document Map

| File | Purpose |
| --- | --- |
| `00-project-summary.md` | High-level game pitch, genre, tone, pillars, and player promise |
| `01-story-world.md` | Full parody story, world setup, factions, tone, and campaign arc |
| `02-level-01.md` | Level 1 breakdown, tutorial flow, objectives, content, and pacing |
| `03-items-structures-resources.md` | Resource systems, items, consumables, structures, and economy |
| `04-troops-allies.md` | Player factions, troop classes, unit examples, roles, and upgrade hooks |
| `05-enemies-bosses.md` | Enemy rosters, rival commanders, elite enemies, and boss concepts |
| `06-game-mechanics.md` | Core gameplay loop, combat systems, timing rules, progression, and balancing principles |
| `07-ui-ux-pages.md` | UI/UX vision, navigation flow, page count, and screen-by-screen requirements |
| `08-technical-spec.md` | Kotlin technical stack, libraries, architecture, asset pipeline, and device specs |
| `fix/README.md` | Fix documentation for Unity 6.5 MCP package compatibility |
| `fix/duplicate-classes.md` | Fix for AurynSky asset pack duplicate class definitions |
| `fix/quick-reference.md` | Quick reference for re-applying fixes |

## Tone Rule

Every doc in this folder follows the same tone target:

- silly
- fun
- parody-first
- no grimdark stakes
- conflict caused by ridiculous people making terrible decisions with total confidence

## Current Product Snapshot

- Genre: lane-based strategy and tower warfare
- Platforms: Android first, desktop second
- Camera: side-view 2D battlefield
- Match length target: 4 to 8 minutes
- Art mood: colorful fantasy nonsense with exaggerated faction identity
- Tech recommendation: Kotlin with libGDX and KTX
